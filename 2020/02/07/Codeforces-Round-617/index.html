Problem A : Array with Odd Sum

稍微想一下就會發現可以分成兩種情況

  1. $n$ 是奇數
    如果陣列 $a$ 裡有至少一個奇數的話，$a$ 的元素總和一定可以被操作成奇數。
  2. $n$ 是偶數
    陣列 $a$ 裡必須同時有奇數和偶數，$a$ 的元素總和才有機會變成奇數。因為偶數個偶數或者奇數不管怎麼換加起來都會是偶數。

Problem C : Yet Another Walking Robot

把每一次移動後的座標算出來，然後檢查之前是否曾經到過這個座標，如果到過的話就檢查是否為最短長度。

設第一步的時候在 $(0,0)$ 的位置，接下來就是按照字串操作。

Problem D : Fight with Monsters

可以發現如果把 $hp[i]$ 減 $1$ 然後模 $(a+b)$，$hp[i] \in [0,a]$，代表不需要 skip；若 $hp[i] \notin [0,a]$，則要 skip 的次數會是 $hp[i] / a$。

因為

$hp[i]$ 跳過次數
$[0,a]$ 0 skip
$[a + 1, 2a]$ 1 skip
… …

然後 sort $hp$ 然後看用 $k$ 次 skip 可以再多殺幾隻怪獸。

Problem E1 : String Coloring (easy version)

可以發現如果 $i < j \ and \ s[j] < s[i]$ 的話，兩個字母必須為不同顏色才能交換。

所以可以在每個這樣的 $i$ 跟 $j$ 之間建邊，然後 dfs 就好了。

Problem E2 : String Coloring (hard version)

可以考慮用 dp 的方式，其實跟上一題的想法差不多。
:::info
若 $i < i + 1 < … < k \ and \ s[i] > s[i+1] > … s[k]$ 則至少需要 $k$ 種顏色。
:::
所以從字串後面做回來，每一次檢查字典序在這個字母前面的字母的 dp 值做更新就可以了。
